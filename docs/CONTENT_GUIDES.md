# Editorial Guides — System Index

Top-level entry point for the `/guides` editorial content system. Use this page to find the right doc for the task at hand.

## What is this?

Static editorial posts (curated city / category / hashtag guides, data recaps) rendered by the web client under `/guides/...`. Posts are JSON files in `therr-client-web/src/content/guides/` bundled into both the client and SSR bundles at build time, generated by the `scripts/generate-content` CLI.

## Where the pieces live

| Concern | Location |
|---------|----------|
| Post JSON (source of truth) | `therr-client-web/src/content/guides/*.json` |
| Post schema | `scripts/generate-content/utils/contentSchema.ts` |
| Frontend loader + lookups | `therr-client-web/src/utilities/guideContent.ts` |
| Route + section renderers | `therr-client-web/src/routes/Guide/` |
| JSON-LD builder | `therr-client-web/src/utilities/guideJsonLd.ts` |
| Sitemap entry | `/sitemap-guides.xml` (in `therr-client-web/src/server-client.tsx`) |
| "Featured in" backlink | `therr-client-web/src/routes/ViewSpace.tsx` |
| CLI scripts | `scripts/generate-content/` (diagnostics in `diagnostics/`) |
| Skill wrapper | `.claude/skills/generate-content/SKILL.md` |

## Common workflows

- **Create a new guide** → use the `generate-content` skill, or drive `scripts/generate-content/save-post.ts` directly
- **Refresh an existing guide** (prune stale spaces, add new ones) → `scripts/generate-content/refresh-post.ts`
- **Find candidate (city, category) buckets** → `scripts/generate-content/discover-categories.ts`
- **Diagnose metric data** (impression names, recency) → `scripts/generate-content/diagnostics/`
