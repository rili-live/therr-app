## Bug found + fixed: transcript-capture.js Stop hook never fired
- The original `.claude/hooks/transcript-capture.js` gated on `input.stop_reason === 'end_turn' && input.response`. **Neither field exists** in the Claude Code Stop hook stdin payload, so the hook silently no-op'd every turn — which is why `context/transcripts/` never got created.
- Real Stop payload: `{ session_id, transcript_path, cwd, hook_event_name, ... }`. The response text is NOT in the payload; you must read it from the JSONL at `transcript_path`.
- Transcript JSONL: each line is an envelope `{ type, uuid, message, ... }`. The final response of a turn is the last envelope with `type === 'assistant'` AND `message.stop_reason === 'end_turn'`; its text is in `message.content[]` blocks of `type === 'text'`. (Tool-only turns have `stop_reason === 'tool_use'` and no text.)
- Rewrote the hook to parse `transcript_path`, grab the last end_turn assistant message's text, append first 500 chars, with a per-session dedupe guard (`.last-captured.json`, gitignored) so repeat Stop fires don't double-append.
- Gotcha when verifying: `grep -c '^## '` over the daily file overcounts because captured responses contain their own `##` markdown headings. Count real entries with `grep -cE '^## [0-9]{2}:[0-9]{2}:[0-9]{2}$'`.
